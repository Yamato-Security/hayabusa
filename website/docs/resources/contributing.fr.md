# Contribution et support

## Contribution

Nous serions ravis de recevoir toute forme de contribution.
Les pull requests, la création de règles et les exemples de journaux evtx sont les bienvenus en priorité, mais les demandes de fonctionnalités, les signalements de bugs, etc. sont également très appréciés.

À tout le moins, si vous aimez notre outil, donnez-nous une étoile sur GitHub et montrez votre soutien !

## Signalement de bugs

Veuillez signaler tout bug que vous trouvez [ici.](https://github.com/Yamato-Security/hayabusa/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
Ce projet est actuellement activement maintenu et nous serons heureux de corriger tout bug signalé.

Si vous rencontrez des problèmes (faux positifs, bugs, etc.) avec les règles Hayabusa, veuillez les signaler sur la page des issues GitHub de hayabusa-rules [ici](https://github.com/Yamato-Security/hayabusa-rules/issues/new).

Si vous rencontrez des problèmes (faux positifs, bugs, etc.) avec les règles Sigma, veuillez les signaler sur la page des issues GitHub en amont de SigmaHQ [ici](https://github.com/SigmaHQ/sigma/issues).

Hayabusa est publié sous [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) et toutes les règles sont publiées sous la [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md).
Vous pouvez utiliser librement Hayabusa en interne, dans des solutions SaaS, pour des travaux de conseil, etc.
Cependant, si vous utilisez Hayabusa dans un type de solution SaaS et y apportez des améliorations, nous vous demandons de rendre ces améliorations open source et de contribuer en retour au projet.

Hayabusa utilise les données GeoLite2 créées par MaxMind, disponibles sur [https://www.maxmind.com](https://www.maxmind.com).
