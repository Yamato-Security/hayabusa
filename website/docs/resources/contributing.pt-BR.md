# Contribuição e Suporte

## Contribuição

Adoraríamos receber qualquer forma de contribuição.
Pull requests, criação de regras e logs evtx de exemplo são os melhores, mas solicitações de recursos, notificações de bugs, etc... também são muito bem-vindas.

No mínimo, se você gostou da nossa ferramenta, dê uma estrela para nós no GitHub e demonstre seu apoio!

## Envio de Bugs

Por favor, envie quaisquer bugs que você encontrar [aqui.](https://github.com/Yamato-Security/hayabusa/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
Este projeto está atualmente em manutenção ativa e ficamos felizes em corrigir quaisquer bugs reportados.

Se você encontrar quaisquer problemas (falsos positivos, bugs, etc...) com as regras do Hayabusa, por favor reporte-os na página de issues do GitHub do hayabusa-rules [aqui](https://github.com/Yamato-Security/hayabusa-rules/issues/new).

Se você encontrar quaisquer problemas (falsos positivos, bugs, etc...) com as regras Sigma, por favor reporte-os na página de issues do GitHub upstream do SigmaHQ [aqui](https://github.com/SigmaHQ/sigma/issues).

## Licença

O Hayabusa é distribuído sob a [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) e todas as regras são distribuídas sob a [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md).
Você pode usar livremente o Hayabusa internamente, em soluções SaaS, para trabalhos de consultoria, etc...
No entanto, se você usar o Hayabusa em algum tipo de solução SaaS e fizer melhorias nele, pedimos que você disponibilize essas melhorias como código aberto e retribua ao projeto.

O Hayabusa usa dados GeoLite2 criados pela MaxMind, disponíveis em [https://www.maxmind.com](https://www.maxmind.com).
