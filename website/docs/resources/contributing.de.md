# Mitwirken & Support

## Mitwirken

Wir freuen uns über jede Form der Mitwirkung.
Pull Requests, das Erstellen von Regeln und Beispiel-evtx-Logs sind am besten, aber auch Funktionswünsche, das Melden von Fehlern usw. sind sehr willkommen.

Wenn Ihnen unser Tool gefällt, geben Sie uns zumindest einen Stern auf GitHub und zeigen Sie Ihre Unterstützung!

## Fehler melden

Bitte melden Sie alle Fehler, die Sie finden, [hier.](https://github.com/Yamato-Security/hayabusa/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
Dieses Projekt wird derzeit aktiv gepflegt und wir beheben gerne alle gemeldeten Fehler.

Wenn Sie Probleme (False Positives, Fehler usw.) mit Hayabusa-Regeln finden, melden Sie diese bitte auf der hayabusa-rules GitHub-Issues-Seite [hier](https://github.com/Yamato-Security/hayabusa-rules/issues/new).

Wenn Sie Probleme (False Positives, Fehler usw.) mit Sigma-Regeln finden, melden Sie diese bitte auf der vorgelagerten SigmaHQ GitHub-Issues-Seite [hier](https://github.com/SigmaHQ/sigma/issues).

## Lizenz

Hayabusa wird unter [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) veröffentlicht und alle Regeln werden unter der [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md) veröffentlicht.
Sie dürfen Hayabusa frei intern, für SaaS-Lösungen, für Beratungstätigkeiten usw. verwenden.
Wenn Sie Hayabusa jedoch in einer Art SaaS-Lösung einsetzen und Verbesserungen daran vornehmen, bitten wir Sie, diese Verbesserungen als Open Source zur Verfügung zu stellen und an das Projekt zurückzugeben.

Hayabusa verwendet GeoLite2-Daten, die von MaxMind erstellt wurden und unter [https://www.maxmind.com](https://www.maxmind.com) verfügbar sind.
