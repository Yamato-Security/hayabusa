# การมีส่วนร่วมและการสนับสนุน

## การมีส่วนร่วม

เรายินดีต้อนรับการมีส่วนร่วมในทุกรูปแบบ
Pull request การสร้างกฎ และตัวอย่าง evtx log นั้นดีที่สุด แต่คำขอฟีเจอร์ การแจ้งบั๊กให้เราทราบ ฯลฯ ก็ได้รับการต้อนรับเช่นกัน

อย่างน้อยที่สุด หากคุณชอบเครื่องมือของเรา โปรดให้ดาวแก่เราบน GitHub และแสดงการสนับสนุนของคุณ!

## การส่งรายงานบั๊ก

โปรดส่งบั๊กใด ๆ ที่คุณพบ [ที่นี่](https://github.com/Yamato-Security/hayabusa/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
โปรเจกต์นี้กำลังได้รับการดูแลอย่างต่อเนื่อง และเรายินดีที่จะแก้ไขบั๊กใด ๆ ที่ได้รับการรายงาน

หากคุณพบปัญหาใด ๆ (false positive บั๊ก ฯลฯ) กับกฎของ Hayabusa โปรดรายงานปัญหาเหล่านั้นไปยังหน้า GitHub issues ของ hayabusa-rules [ที่นี่](https://github.com/Yamato-Security/hayabusa-rules/issues/new)

หากคุณพบปัญหาใด ๆ (false positive บั๊ก ฯลฯ) กับกฎของ Sigma โปรดรายงานปัญหาเหล่านั้นไปยังหน้า GitHub issues ต้นทางของ SigmaHQ [ที่นี่](https://github.com/SigmaHQ/sigma/issues)

## สัญญาอนุญาต

Hayabusa เผยแพร่ภายใต้ [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) และกฎทั้งหมดเผยแพร่ภายใต้ [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md)
คุณสามารถใช้ Hayabusa ภายในองค์กร โซลูชัน SaaS สำหรับงานที่ปรึกษา ฯลฯ ได้อย่างอิสระ
อย่างไรก็ตาม หากคุณใช้ Hayabusa ในโซลูชันประเภท SaaS และทำการปรับปรุงมัน เราขอให้คุณเปิดเผยซอร์สโค้ดของการปรับปรุงเหล่านั้นและตอบแทนกลับสู่โปรเจกต์

Hayabusa ใช้ข้อมูล GeoLite2 ที่สร้างโดย MaxMind ซึ่งมีให้บริการจาก [https://www.maxmind.com](https://www.maxmind.com)
