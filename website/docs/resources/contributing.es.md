# Contribución y soporte

## Contribución

Nos encantaría recibir cualquier forma de contribución.
Las pull requests, la creación de reglas y los registros evtx de muestra son lo mejor, pero las solicitudes de funciones, notificarnos de errores, etc... también son muy bienvenidas.

Como mínimo, si te gusta nuestra herramienta, ¡por favor danos una estrella en GitHub y muestra tu apoyo!

## Envío de errores

Por favor, envía cualquier error que encuentres [aquí.](https://github.com/Yamato-Security/hayabusa/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
Este proyecto se mantiene actualmente de forma activa y estaremos encantados de corregir cualquier error reportado.

Si encuentras cualquier problema (falsos positivos, errores, etc...) con las reglas de Hayabusa, por favor repórtalo en la página de issues de GitHub de hayabusa-rules [aquí](https://github.com/Yamato-Security/hayabusa-rules/issues/new).

Si encuentras cualquier problema (falsos positivos, errores, etc...) con las reglas de Sigma, por favor repórtalo en la página de issues de GitHub upstream de SigmaHQ [aquí](https://github.com/SigmaHQ/sigma/issues).

## Licencia

Hayabusa se publica bajo [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) y todas las reglas se publican bajo la [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md).
Puedes usar Hayabusa libremente de forma interna, en soluciones SaaS, para trabajos de consultoría, etc...
Sin embargo, si usas Hayabusa en un tipo de solución SaaS y le haces mejoras, te pedimos que liberes el código de esas mejoras y las devuelvas al proyecto.

Hayabusa utiliza datos de GeoLite2 creados por MaxMind, disponibles en [https://www.maxmind.com](https://www.maxmind.com).
