# المساهمة والدعم

## المساهمة

نرحب بأي شكل من أشكال المساهمة.
طلبات السحب (Pull requests) وإنشاء القواعد وعينات سجلات evtx هي الأفضل، ولكن طلبات الميزات والإبلاغ عن الأخطاء وما إلى ذلك... مرحب بها أيضًا.

على الأقل، إذا أعجبتك أداتنا فمن فضلك امنحنا نجمة على GitHub وأظهر دعمك!

## الإبلاغ عن الأخطاء

يُرجى الإبلاغ عن أي أخطاء تجدها [هنا.](https://github.com/Yamato-Security/hayabusa/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
يخضع هذا المشروع حاليًا للصيانة النشطة ويسعدنا إصلاح أي أخطاء يتم الإبلاغ عنها.

إذا وجدت أي مشكلات (نتائج إيجابية خاطئة، أخطاء، وما إلى ذلك...) في قواعد Hayabusa، فيُرجى الإبلاغ عنها إلى صفحة مشكلات hayabusa-rules على GitHub [هنا](https://github.com/Yamato-Security/hayabusa-rules/issues/new).

إذا وجدت أي مشكلات (نتائج إيجابية خاطئة، أخطاء، وما إلى ذلك...) في قواعد Sigma، فيُرجى الإبلاغ عنها إلى صفحة مشكلات SigmaHQ الأصلية على GitHub [هنا](https://github.com/SigmaHQ/sigma/issues).

## الترخيص

يُصدر Hayabusa بموجب [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) وتُصدر جميع القواعد بموجب [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md).
يمكنك استخدام Hayabusa بحرية داخليًا، وفي حلول SaaS، وللأعمال الاستشارية، وما إلى ذلك...
ومع ذلك، إذا استخدمت Hayabusa في نوع من حلول SaaS وأجريت تحسينات عليه، فإننا نطلب منك جعل تلك التحسينات مفتوحة المصدر وإعادتها إلى المشروع.

يستخدم Hayabusa بيانات GeoLite2 التي أنشأتها MaxMind، والمتوفرة من [https://www.maxmind.com](https://www.maxmind.com).
