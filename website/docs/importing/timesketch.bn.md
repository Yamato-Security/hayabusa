# Timesketch দিয়ে Hayabusa ফলাফল বিশ্লেষণ

## সম্পর্কে

"[Timesketch](https://timesketch.org/) হলো সহযোগিতামূলক ফরেনসিক টাইমলাইন বিশ্লেষণের জন্য একটি ওপেন-সোর্স টুল। sketch ব্যবহার করে আপনি এবং আপনার সহযোগীরা সহজেই আপনাদের টাইমলাইনগুলো সাজাতে পারেন এবং একই সাথে সেগুলো বিশ্লেষণ করতে পারেন। সমৃদ্ধ অ্যানোটেশন, মন্তব্য, ট্যাগ এবং স্টার দিয়ে আপনার কাঁচা ডেটায় অর্থ যোগ করুন।"

ছোট তদন্তের ক্ষেত্রে যেখানে আপনি মাত্র কয়েকশ MB আকারের একটি CSV ফাইল বিশ্লেষণ করছেন এবং একা কাজ করছেন, সেক্ষেত্রে Timeline Explorer উপযুক্ত, তবে যখন আপনি বৃহত্তর ডেটা নিয়ে বা একটি দলের সাথে কাজ করছেন, তখন Timesketch এর মতো একটি টুল অনেক ভালো।

Timesketch নিম্নলিখিত সুবিধাগুলো প্রদান করে:
1. এটি খুব দ্রুত এবং বৃহৎ ডেটা পরিচালনা করতে পারে
2. এটি একটি সহযোগিতামূলক টুল যেখানে একাধিক ব্যবহারকারী একই সাথে এটি ব্যবহার করতে পারেন
3. এটি উন্নত ডেটা বিশ্লেষণ, হিস্টোগ্রাম এবং ভিজ্যুয়ালাইজেশন প্রদান করে
4. এটি Windows-এ সীমাবদ্ধ নয়
5. এটি উন্নত কোয়েরি সমর্থন করে

CTI সমর্থন, বিভিন্ন বিশ্লেষক, ইন্টারঅ্যাক্টিভ নোটবুক ইত্যাদির মতো আরও অনেক সুবিধা রয়েছে...
আরও তথ্যের জন্য অনুগ্রহ করে [ব্যবহারকারীর নির্দেশিকা](https://timesketch.org/guides/user/upload-data/) এবং [YouTube চ্যানেল](https://www.youtube.com/channel/UC_n6mMb0OxWRk7xiqiOOcRQ) দেখুন।

একমাত্র অসুবিধা হলো আপনাকে আপনার ল্যাব পরিবেশে একটি Timesketch সার্ভার সেটআপ করতে হবে কিন্তু সৌভাগ্যবশত এটি করা খুবই সহজ।

## ইনস্টল করা
### Docker
[এখানে](https://docs.docker.com/compose/install) অফিসিয়াল নির্দেশাবলী অনুসরণ করুন।

### Ubuntu
**দ্রষ্টব্য:** এগিয়ে যাওয়ার আগে Docker অবশ্যই ইনস্টল করা থাকতে হবে। যদি আপনি ইতিমধ্যে Docker ইনস্টল না করে থাকেন তবে অনুগ্রহ করে উপরের [Docker ইনস্টলেশন নির্দেশাবলী](#docker) অনুসরণ করুন।
আমরা কমপক্ষে 8GB মেমরি সহ সর্বশেষ Ubuntu LTS Server সংস্করণ ব্যবহার করার পরামর্শ দিই।
আপনি এটি [এখানে](https://ubuntu.com/download/server) ডাউনলোড করতে পারেন।
এটি সেটআপ করার সময় minimal install বেছে নিন।
OS সেটআপ করার সময় docker ইনস্টল করবেন না।
আপনার কাছে `ifconfig` উপলব্ধ থাকবে না, তাই এটি `sudo apt install net-tools` দিয়ে ইনস্টল করুন।

এরপর, VM-এর IP ঠিকানা খুঁজে পেতে `ifconfig` চালান এবং ঐচ্ছিকভাবে এতে ssh করুন।

নিম্নলিখিত কমান্ডগুলো চালান:
``` bash
curl -s -O https://raw.githubusercontent.com/google/timesketch/master/contrib/deploy_timesketch.sh
chmod 755 deploy_timesketch.sh
cd /opt
sudo ~/deploy_timesketch.sh
cd timesketch
sudo docker compose up -d

# Create a user named user. Set the password here.
sudo docker compose exec timesketch-web tsctl create-user user
```
### macOS
**দ্রষ্টব্য:** এগিয়ে যাওয়ার আগে, নিশ্চিত করুন যে আপনার সিস্টেমে [Docker Desktop for Mac](https://docs.docker.com/desktop/install/mac/) ইনস্টল করা এবং চলমান আছে।
Timesketch রিপোজিটরি ক্লোন করুন এবং ডিরেক্টরিতে প্রবেশ করুন।
```bash
git clone https://github.com/google/timesketch.git
cd timesketch
```
নিচের ধাপগুলো অনুসরণ করে Docker কন্টেইনার শুরু করুন।
- https://github.com/google/timesketch/tree/master/docker/e2e#build-and-start-containers

## লগ ইন করা

`ifconfig` দিয়ে Timesketch সার্ভারের IP ঠিকানা খুঁজে বের করুন এবং একটি ওয়েব ব্রাউজার দিয়ে এটি খুলুন।
আপনাকে একটি লগইন পৃষ্ঠায় রিডাইরেক্ট করা হবে।
একজন ব্যবহারকারী যোগ করার সময় আপনি যে ব্যবহারকারী শংসাপত্র ব্যবহার করেছিলেন তা দিয়ে লগ ইন করুন।

## একটি নতুন sketch তৈরি করা

`Start a new investigation`-এর অধীনে, `BLANK SKETCH`-এ ক্লিক করুন।
আপনার তদন্তের সাথে প্রাসঙ্গিক কিছু নাম দিয়ে sketch-টির নামকরণ করুন।

## আপনার টাইমলাইন আপলোড করা

`+ ADD TIMELINE`-এ ক্লিক করার পর, আপনি একটি ডায়ালগ বক্স দেখতে পাবেন যা আপনাকে একটি Plaso, JSONL বা CSV ফাইল আপলোড করতে বলবে।
দুর্ভাগ্যবশত, Timesketch বর্তমানে Hayabusa-এর `JSONL` ফরম্যাট ইম্পোর্ট করতে পারে না, তাই নিম্নলিখিত কমান্ড দিয়ে একটি CSV টাইমলাইন তৈরি এবং আপলোড করুন:

```shell
hayabusa-x.x.x-win-x64.exe csv-timeline -d <DIR> -o timesketch-import.csv -p timesketch-verbose --ISO-8601
```

> দ্রষ্টব্য: একটি `timesketch*` প্রোফাইল বেছে নেওয়া এবং UTC-এর জন্য `--ISO-8601` অথবা স্থানীয় সময়ের জন্য `--RFC-3339` হিসাবে timestamp নির্দিষ্ট করা প্রয়োজন। আপনি চাইলে অন্যান্য Hayabusa অপশন যোগ করতে পারেন, তবে `-M, --multiline` অপশন যোগ করবেন না কারণ নতুন লাইনের অক্ষরগুলো ইম্পোর্টকে নষ্ট করে দেবে।

"Select file to upload" ডায়ালগ বক্সে, আপনার টাইমলাইনের নাম `hayabusa`-এর মতো কিছু দিন, `Comma (,)` CSV ডিলিমিটার বেছে নিন এবং `SUBMIT`-এ ক্লিক করুন।

> যদি আপনার CSV ফাইল আপলোড করার জন্য খুব বড় হয়, তাহলে আপনি Takajo-এর [split-csv-timeline](https://github.com/Yamato-Security/takajo?tab=readme-ov-file#split-csv-timeline-command) কমান্ড দিয়ে ফাইলটিকে একাধিক CSV ফাইলে ভাগ করতে পারেন।

ফাইলটি ইম্পোর্ট হওয়ার সময় আপনি একটি ঘূর্ণায়মান বৃত্ত দেখতে পাবেন তাই অনুগ্রহ করে এটি শেষ না হওয়া পর্যন্ত এবং `hayabusa` দেখানো পর্যন্ত অপেক্ষা করুন।

## বিশ্লেষণের টিপস

### টাইমলাইন দেখানো

**দ্রষ্টব্য: ইম্পোর্ট সফলভাবে শেষ হওয়ার পরেও, এটি `Your search did not match any events` দেখাবে এবং `hayabusa` টাইমলাইনে `0` টি ইভেন্ট থাকবে।**

`*` অনুসন্ধান করুন এবং নিচে দেখানো হিসাবে ইভেন্টগুলো প্রদর্শিত হবে:

![Timesketch results](../assets/doc/TimesketchImport/TimesketchResults.png)

### অ্যালার্টের বিবরণ

আপনি যদি `message` কলামের অধীনে একটি অ্যালার্ট নিয়মের শিরোনামে ক্লিক করেন, তাহলে আপনি অ্যালার্ট সম্পর্কে বিস্তারিত তথ্য পাবেন:

![Alert details](../assets/doc/TimesketchImport/AlertDetails.png)

আপনি যদি sigma নিয়মের যুক্তি বুঝতে চান, বিবরণ ও রেফারেন্স ইত্যাদি দেখতে চান... অনুগ্রহ করে [hayabusa-rules](https://github.com/Yamato-Security/hayabusa-rules) রিপোজিটরিতে নিয়মটি খুঁজে দেখুন।

#### ফিল্ড ফিল্টারিং

একটি ইভেন্টের নিয়মের শিরোনামে ক্লিক করে এর বিবরণ খোলার পর আপনি যেকোনো ফিল্ডের উপর হোভার করে সহজেই মানটি ফিল্টার ইন বা আউট করতে পারেন:

![Filter In Out](../assets/doc/TimesketchImport/FilterInOut.png)

#### অ্যাগ্রিগেশন অ্যানালিটিক্স

হোভার করার সময়, যদি আপনি বাম দিকের `Aggregation dialog` আইকনে ক্লিক করেন, তাহলে আপনি সেই ফিল্ড সম্পর্কে সত্যিই দুর্দান্ত ইভেন্ট ডেটা অ্যানালিটিক্স পাবেন:

![Event Data Analytics](../assets/doc/TimesketchImport/EventDataAnalytics.png)

#### ব্যবহারকারীর মন্তব্য

আপনি যখন বিস্তারিত তথ্য পেতে একটি অ্যালার্টে ক্লিক করেন, তখন ডান পাশে একটি নতুন মন্তব্য ডায়ালগ বক্স আইকন দেখানো হয়, নিচে যেমন দেখানো হয়েছে:

![Comment Icon](../assets/doc/TimesketchImport/CommentIcon.png)

এখানে, ব্যবহারকারীরা একটি চ্যাট শুরু করতে এবং তদন্ত সম্পর্কে মন্তব্য লিখতে পারেন।

> আপনি যদি একটি দলে কাজ করেন, তাহলে আপনার সম্ভবত প্রতিটি সদস্যের জন্য আলাদা ব্যবহারকারী অ্যাকাউন্ট তৈরি করা উচিত যাতে আপনি জানতে পারেন কে কী লিখেছে।

![Comment chat](../assets/doc/TimesketchImport/CommentChat.png)

> আপনি যদি একটি মন্তব্যের উপর হোভার করেন, আপনি সহজেই বার্তাগুলো সম্পাদনা ও মুছে ফেলতে পারেন।

### কলাম পরিবর্তন করা

ডিফল্টভাবে, শুধুমাত্র timestamp এবং অ্যালার্ট নিয়মের শিরোনাম প্রদর্শিত হবে তাই ফিল্ডগুলো কাস্টমাইজ করতে `Modify columns` আইকনে ক্লিক করুন:

![ModifyColumnsIcon](../assets/doc/TimesketchImport/ModifyColumnsIcon.png)

এটি নিম্নলিখিত ডায়ালগ বক্সটি খুলবে:

![Select columns](../assets/doc/TimesketchImport/SelectColumns.png)

আমরা কমপক্ষে নিম্নলিখিত কলামগুলো **ক্রমানুসারে** যোগ করার পরামর্শ দিই:

1. `Level`
2. `Computer`
3. `Channel`
4. `EventID`
5. `RecordID`

কলামগুলোর ক্রম আপনি যে ক্রমে সেগুলো যোগ করেন তার উপর নির্ভর করে পরিবর্তিত হবে, তাই আগে আরও গুরুত্বপূর্ণ ফিল্ডগুলো যোগ করুন।

আপনার স্ক্রিনে যদি এখনও জায়গা থাকে, আমরা এখানে যেমন দেখানো হয়েছে, `Details` যোগ করারও পরামর্শ দিই:

![Details](../assets/doc/TimesketchImport/Details.png)

আপনার স্ক্রিনে যদি এখনও জায়গা থাকে, আমরা `ExtraFieldInfo` যোগ করারও পরামর্শ দিই, তবে, আপনি এখানে যেমন দেখছেন, আপনি যদি খুব বেশি কলাম যোগ করেন তাহলে `message` ফিল্ডটি খুব সরু হয়ে যাবে এবং আপনি আর অ্যালার্টের শিরোনামগুলো পড়তে পারবেন না:

![Too much details](../assets/doc/TimesketchImport/TooMuchDetails.png)

আপনি যদি একটি বারে ক্লিক করেন, এটি একটি সময় ফিল্টার তৈরি করবে যা শুধুমাত্র সেই সময়কালের ফলাফল দেখাবে।

#### বর্তমান অনুসন্ধান সংরক্ষণ

আপনি যদি timestamp-এর ঠিক উপরে এবং `Toggle Event Histogram` আইকনের বাম পাশে থাকা `Save current search` আইকনে ক্লিক করেন, তাহলে আপনি আপনার বর্তমান অনুসন্ধান কোয়েরির পাশাপাশি কলাম কনফিগারেশন `Saved Searches`-এ সংরক্ষণ করতে পারেন।
পরে, বাম পাশের সাইডবার থেকে আপনি সহজেই আপনার প্রিয় অনুসন্ধানগুলো অ্যাক্সেস করতে পারেন।

### সার্চ বার

শুধুমাত্র নির্দিষ্ট তীব্রতা স্তরের অ্যালার্ট দেখিয়ে শুরু করার জন্য এখানে কিছু সহজ কোয়েরি দেওয়া হলো:
1. শুধুমাত্র critical অ্যালার্ট দেখানোর জন্য `Level:crit`।
2. high এবং critical অ্যালার্ট দেখানোর জন্য `Level:crit OR Level:high`
3. informational অ্যালার্ট লুকানোর জন্য `NOT Level:info`

আপনি ফিল্ডের নাম প্লাস `:` প্লাস মান টাইপ করে সহজেই ফিল্টার করতে পারেন।
আপনি `AND`, `OR`, এবং `NOT` দিয়ে ফিল্টার একত্রিত করতে পারেন।
Wildcard এবং regular expression সমর্থিত।

আরও উন্নত কোয়েরির জন্য [এখানে](https://timesketch.org/guides/user/search-query-guide/) ব্যবহারকারীর নির্দেশিকা দেখুন।

#### অনুসন্ধান ইতিহাস

আপনি যদি সার্চ বারের বাম পাশে থাকা ঘড়ির আইকনে ক্লিক করেন তাহলে আপনি পূর্বে প্রবেশ করানো কোয়েরিগুলো দেখাতে পারেন।
আপনি পূর্ববর্তী এবং পরবর্তী কোয়েরি চালাতে বাম এবং ডান তীর আইকনেও ক্লিক করতে পারেন।

### উল্লম্ব elipsis

আপনি যদি একটি timestamp-এর বাম পাশে থাকা উল্লম্ব elipsis-এ ক্লিক করেন এবং `Context search`-এ ক্লিক করেন, তাহলে আপনি একটি নির্দিষ্ট ইভেন্টের আগে এবং পরে ঘটে যাওয়া অ্যালার্টগুলো দেখতে পারেন:

![Vertical elipsis](../assets/doc/TimesketchImport/VerticalElipsisContext.png)

এটি নিচেরটি নিয়ে আসবে:

![Context Search](../assets/doc/TimesketchImport/ContextSearch.png)

উপরের উদাহরণে, 60 সেকেন্ড (`60S`) এর আগে এবং পরের ইভেন্টগুলো দেখানো হচ্ছে কিন্তু আপনি এটি +- 1 সেকেন্ড (`1S`) থেকে +- 60 মিনিট (`60M`) পর্যন্ত সামঞ্জস্য করতে পারেন।

আপনি যদি দেখানো ইভেন্টগুলোতে আরও গভীরে যেতে চান, তাহলে স্ট্যান্ডার্ড টাইমলাইনে ইভেন্টগুলো দেখানোর জন্য `Replace Search`-এ ক্লিক করুন।

### স্টার এবং ট্যাগ

আপনি একটি timestamp-এর বাম পাশে থাকা স্টার আইকনে ক্লিক করে এটিকে স্টার দিতে পারেন এবং একটি গুরুত্বপূর্ণ ইভেন্ট হিসাবে চিহ্নিত করতে পারেন।

আপনি ইভেন্টগুলোতে ট্যাগও যোগ করতে পারেন।
এটি অন্যদের কাছে ইঙ্গিত দিতে কার্যকর যে আপনি নিশ্চিত করেছেন যে একটি ইভেন্ট সন্দেহজনক, ক্ষতিকারক, একটি false positive ইত্যাদি...
আপনি যদি একটি দলে কাজ করেন, তাহলে আপনি `under investigation by xxx` এর মতো ট্যাগ তৈরি করতে পারেন যাতে ইঙ্গিত দেওয়া যায় যে কেউ বর্তমানে অ্যালার্টটি তদন্ত করছে।
