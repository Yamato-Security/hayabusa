# কমান্ড তালিকা

## বিশ্লেষণ কমান্ডসমূহ:
* `computer-metrics`: কম্পিউটারের নামের উপর ভিত্তি করে ইভেন্টের সংখ্যা প্রিন্ট করে।
* `eid-metrics`: Event ID-এর উপর ভিত্তি করে ইভেন্টের সংখ্যা এবং শতাংশ প্রিন্ট করে।
* `expand-list`: `rules` ফোল্ডার থেকে `expand` প্লেসহোল্ডার এক্সট্র্যাক্ট করে।
* `extract-base64`: ইভেন্ট থেকে base64 স্ট্রিং এক্সট্র্যাক্ট এবং ডিকোড করে।
* `log-metrics`: লগ ফাইল মেট্রিক্স প্রিন্ট করে।
* `logon-summary`: লগন ইভেন্টের একটি সারসংক্ষেপ প্রিন্ট করে।
* `pivot-keywords-list`: পিভট করার জন্য সন্দেহজনক কীওয়ার্ডের একটি তালিকা প্রিন্ট করে।
* `search`: কীওয়ার্ড বা রেগুলার এক্সপ্রেশন দিয়ে সমস্ত ইভেন্ট অনুসন্ধান করে

## কনফিগ কমান্ডসমূহ:
* `config-critical-systems`: ডোমেইন কন্ট্রোলার এবং ফাইল সার্ভারের মতো গুরুত্বপূর্ণ সিস্টেম খুঁজে বের করে।

## DFIR টাইমলাইন কমান্ডসমূহ:
* `csv-timeline`: CSV ফরম্যাটে টাইমলাইন সংরক্ষণ করে।
* `json-timeline`: JSON/JSONL ফরম্যাটে টাইমলাইন সংরক্ষণ করে।
* `level-tuning`: অ্যালার্টের `level` কাস্টম টিউন করে।
* `list-profiles`: উপলব্ধ আউটপুট প্রোফাইলগুলি তালিকাভুক্ত করে।
* `set-default-profile`: ডিফল্ট প্রোফাইল পরিবর্তন করে।
* `update-rules`: [hayabusa-rules](https://github.com/Yamato-Security/hayabusa-rules) GitHub রিপোজিটরির সর্বশেষ নিয়মের সাথে নিয়মগুলি সিঙ্ক করে।

## সাধারণ কমান্ডসমূহ:
* `help`: এই বার্তা বা প্রদত্ত সাবকমান্ড(সমূহ)-এর সাহায্য প্রিন্ট করে
* `list-contributors`: অবদানকারীদের তালিকা প্রিন্ট করে
