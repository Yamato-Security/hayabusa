# মূল লক্ষ্যসমূহ

## থ্রেট হান্টিং এবং এন্টারপ্রাইজ-ব্যাপী DFIR

Hayabusa-তে বর্তমানে ৪০০০-এর বেশি Sigma রুল এবং ১৭০-এর বেশি Hayabusa অন্তর্নির্মিত ডিটেকশন রুল রয়েছে এবং নিয়মিতভাবে আরও রুল যুক্ত হচ্ছে।
এটি [Velociraptor](https://docs.velociraptor.app/)-এর [Hayabusa artifact](https://docs.velociraptor.app/exchange/artifacts/pages/windows.eventlogs.hayabusa/)-এর সাহায্যে বিনামূল্যে এন্টারপ্রাইজ-ব্যাপী প্রোঅ্যাক্টিভ থ্রেট হান্টিং পাশাপাশি DFIR (Digital Forensics and Incident Response)-এর জন্য ব্যবহার করা যেতে পারে।
এই দুটি ওপেন-সোর্স টুল একত্রিত করে, পরিবেশে কোনো SIEM সেটআপ না থাকলে আপনি মূলত পূর্ববর্তীভাবে একটি SIEM পুনরায় তৈরি করতে পারেন।
আপনি [Eric Capuano](https://twitter.com/eric_capuano)-এর Velociraptor ওয়াকথ্রু [এখানে](https://www.youtube.com/watch?v=Q1IoGX--814) দেখে এটি কীভাবে করতে হয় তা শিখতে পারেন।

## দ্রুত ফরেনসিক টাইমলাইন তৈরি

Windows ইভেন্ট লগ বিশ্লেষণ ঐতিহ্যগতভাবে একটি অত্যন্ত দীর্ঘ এবং ক্লান্তিকর প্রক্রিয়া হয়ে থাকে কারণ Windows ইভেন্ট লগ ১) এমন একটি ডেটা ফরম্যাটে থাকে যা বিশ্লেষণ করা কঠিন এবং ২) বেশিরভাগ ডেটা হলো নয়েজ এবং তদন্তের জন্য উপযোগী নয়।
Hayabusa-এর লক্ষ্য হলো কেবলমাত্র উপযোগী ডেটা বের করা এবং তা যথাসম্ভব সংক্ষিপ্ত, সহজে পড়ার মতো একটি ফরম্যাটে উপস্থাপন করা যা কেবল পেশাদারভাবে প্রশিক্ষিত বিশ্লেষকদের জন্যই নয়, বরং যেকোনো Windows সিস্টেম প্রশাসকের জন্যও ব্যবহারযোগ্য।
Hayabusa আশা করে যে বিশ্লেষকরা ঐতিহ্যগত Windows ইভেন্ট লগ বিশ্লেষণের তুলনায় তাদের কাজের ৮০% ২০% সময়ে সম্পন্ন করতে পারবেন।
