# Über Hayabusa

Hayabusa ist ein **schneller forensischer Zeitleisten-Generator für Windows-Ereignisprotokolle** und ein **Threat-Hunting-Werkzeug**, das von der [Yamato Security](https://yamatosecurity.connpass.com/)-Gruppe in Japan entwickelt wurde.
Hayabusa bedeutet auf Japanisch ["Wanderfalke"](https://en.wikipedia.org/wiki/Peregrine_falcon) und wurde gewählt, weil Wanderfalken die schnellsten Tiere der Welt sind, hervorragend jagen und sich sehr gut trainieren lassen.
Es ist in dem speichersicheren [Rust](https://www.rust-lang.org/) geschrieben, unterstützt Multithreading, um so schnell wie möglich zu sein, und ist das einzige Open-Source-Werkzeug, das die Sigma-Spezifikation einschließlich der v2-Korrelationsregeln vollständig unterstützt.
Hayabusa kann das Parsen von [Upstream-Sigma](https://github.com/SigmaHQ/sigma)-Regeln verarbeiten, jedoch werden die Sigma-Regeln, die wir verwenden und im [hayabusa-rules-Repository](https://github.com/Yamato-Security/hayabusa-rules) hosten, einer gewissen Konvertierung unterzogen, um das Laden der Regeln flexibler zu gestalten und Fehlalarme zu reduzieren.
Details dazu können Sie in der README-Datei des [sigma-to-hayabusa-converter-Repositorys](https://github.com/Yamato-Security/sigma-to-hayabusa-converter) nachlesen.
Hayabusa kann entweder auf einzelnen laufenden Systemen zur Live-Analyse ausgeführt werden, durch das Sammeln von Protokollen von einzelnen oder mehreren Systemen zur Offline-Analyse oder durch das Ausführen des [Hayabusa-Artefakts](https://docs.velociraptor.app/exchange/artifacts/pages/windows.eventlogs.hayabusa/) mit [Velociraptor](https://docs.velociraptor.app/) für unternehmensweites Threat Hunting und Incident Response.
Die Ausgabe wird in einer einzigen CSV/JSON/JSONL-Zeitleiste konsolidiert, um eine einfache Analyse in [LibreOffice](https://www.libreoffice.org/), [Timeline Explorer](https://ericzimmerman.github.io/#!index.md) [Elastic Stack](../importing/elastic-stack.md), [Timesketch](https://timesketch.org/) usw. zu ermöglichen ...
