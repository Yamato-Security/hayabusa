# Клонування через Git

Ви можете виконати `git clone` репозиторію за допомогою наступної команди та скомпілювати бінарний файл із вихідного коду:

**Попередження:** Гілка main репозиторію призначена для розробки, тому ви можете отримати доступ до нових функцій, які ще не вийшли офіційно, проте в ній можуть бути помилки, тож вважайте її нестабільною.

```bash
git clone https://github.com/Yamato-Security/hayabusa.git --recursive
```

> **Примітка:** Якщо ви забудете використати опцію --recursive, теку `rules`, яка керується як git-субмодуль, не буде клоновано.

Ви можете синхронізувати теку `rules` та отримати найновіші правила Hayabusa за допомогою `git pull --recurse-submodules` або скористатися наступною командою:

```bash
hayabusa.exe update-rules
```

Якщо оновлення не вдасться, можливо, вам доведеться перейменувати теку `rules` та спробувати знову.

>> Увага: Під час оновлення правила та файли конфігурації в теці `rules` замінюються найновішими правилами та файлами конфігурації з репозиторію [hayabusa-rules](https://github.com/Yamato-Security/hayabusa-rules).
>> Будь-які зміни, які ви внесли до наявних файлів, буде перезаписано, тому ми рекомендуємо робити резервні копії будь-яких файлів, які ви редагуєте, перед оновленням.
>> Якщо ви виконуєте налаштування рівнів за допомогою `level-tuning`, будь ласка, повторно налаштуйте свої файли правил після кожного оновлення.
>> Якщо ви додаєте **нові** правила всередині теки `rules`, їх **не** буде перезаписано чи видалено під час оновлення.
