# Git Cloning

आप निम्नलिखित कमांड के साथ रिपॉजिटरी को `git clone` कर सकते हैं और स्रोत कोड से बाइनरी कंपाइल कर सकते हैं:

**Warning:** रिपॉजिटरी की मुख्य ब्रांच विकास उद्देश्यों के लिए है इसलिए आप उन नई सुविधाओं तक पहुंच सकते हैं जो अभी तक आधिकारिक रूप से जारी नहीं की गई हैं, हालांकि, इसमें बग हो सकते हैं इसलिए इसे अस्थिर मानें।

```bash
git clone https://github.com/Yamato-Security/hayabusa.git --recursive
```

> **Note:** यदि आप --recursive विकल्प का उपयोग करना भूल जाते हैं, तो `rules` फ़ोल्डर, जिसे एक git सबमॉड्यूल के रूप में प्रबंधित किया जाता है, क्लोन नहीं होगा।

आप `rules` फ़ोल्डर को सिंक कर सकते हैं और `git pull --recurse-submodules` के साथ नवीनतम Hayabusa नियम प्राप्त कर सकते हैं या निम्नलिखित कमांड का उपयोग कर सकते हैं:

```bash
hayabusa.exe update-rules
```

यदि अपडेट विफल हो जाता है, तो आपको `rules` फ़ोल्डर का नाम बदलना और फिर से प्रयास करना पड़ सकता है।

>> Caution: अपडेट करते समय, `rules` फ़ोल्डर में नियम और कॉन्फिग फ़ाइलें [hayabusa-rules](https://github.com/Yamato-Security/hayabusa-rules) रिपॉजिटरी में नवीनतम नियमों और कॉन्फिग फ़ाइलों से बदल दी जाती हैं।
>> आप मौजूदा फ़ाइलों में जो भी परिवर्तन करते हैं वे अधिलेखित हो जाएंगे, इसलिए हम अनुशंसा करते हैं कि अपडेट करने से पहले आप उन फ़ाइलों का बैकअप बनाएं जिन्हें आप संपादित करते हैं।
>> यदि आप `level-tuning` के साथ स्तर ट्यूनिंग कर रहे हैं, तो कृपया प्रत्येक अपडेट के बाद अपनी नियम फ़ाइलों को फिर से ट्यून करें।
>> यदि आप `rules` फ़ोल्डर के अंदर **नए** नियम जोड़ते हैं, तो अपडेट करते समय उन्हें अधिलेखित या हटाया **नहीं** जाएगा।
