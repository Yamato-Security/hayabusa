# Git Cloning

আপনি নিম্নলিখিত কমান্ড দিয়ে রিপোজিটরিটি `git clone` করতে এবং সোর্স কোড থেকে বাইনারি কম্পাইল করতে পারেন:

**সতর্কতা:** রিপোজিটরির মেইন ব্রাঞ্চ উন্নয়নের উদ্দেশ্যে, তাই আপনি এমন নতুন ফিচার ব্যবহার করতে পারবেন যা এখনও আনুষ্ঠানিকভাবে প্রকাশিত হয়নি, তবে এতে বাগ থাকতে পারে তাই এটিকে অস্থিতিশীল হিসেবে বিবেচনা করুন।

```bash
git clone https://github.com/Yamato-Security/hayabusa.git --recursive
```

> **নোট:** আপনি যদি --recursive অপশন ব্যবহার করতে ভুলে যান, তাহলে `rules` ফোল্ডার, যা একটি git submodule হিসেবে পরিচালিত হয়, ক্লোন হবে না।

আপনি `git pull --recurse-submodules` দিয়ে `rules` ফোল্ডার সিঙ্ক করতে এবং সর্বশেষ Hayabusa নিয়মাবলী পেতে পারেন অথবা নিম্নলিখিত কমান্ড ব্যবহার করতে পারেন:

```bash
hayabusa.exe update-rules
```

আপডেট ব্যর্থ হলে, আপনাকে `rules` ফোল্ডারের নাম পরিবর্তন করে আবার চেষ্টা করতে হতে পারে।

>> সতর্কতা: আপডেট করার সময়, `rules` ফোল্ডারের নিয়মাবলী এবং কনফিগ ফাইলগুলি [hayabusa-rules](https://github.com/Yamato-Security/hayabusa-rules) রিপোজিটরির সর্বশেষ নিয়মাবলী এবং কনফিগ ফাইল দিয়ে প্রতিস্থাপিত হয়।
>> বিদ্যমান ফাইলগুলিতে আপনি যে কোনো পরিবর্তন করেন তা ওভাররাইট হয়ে যাবে, তাই আমরা সুপারিশ করি যে আপডেট করার আগে আপনি যে ফাইলগুলি সম্পাদনা করেন তার ব্যাকআপ তৈরি করুন।
>> আপনি যদি `level-tuning` দিয়ে লেভেল টিউনিং করেন, তাহলে অনুগ্রহ করে প্রতিটি আপডেটের পরে আপনার নিয়ম ফাইলগুলি পুনরায় টিউন করুন।
>> আপনি যদি `rules` ফোল্ডারের ভিতরে **নতুন** নিয়ম যোগ করেন, তাহলে আপডেট করার সময় সেগুলি ওভাররাইট বা মুছে ফেলা হবে **না**।
