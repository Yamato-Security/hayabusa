# Hayabusa চালানো

## সতর্কতা: অ্যান্টি-ভাইরাস/EDR সতর্কবার্তা এবং ধীর রানটাইম

hayabusa চালানোর চেষ্টা করার সময় অথবা এমনকি শুধু `.yml` রুলগুলি ডাউনলোড করার সময়ও আপনি অ্যান্টি-ভাইরাস বা EDR পণ্য থেকে একটি সতর্কবার্তা পেতে পারেন, কারণ ডিটেকশন সিগনেচারে `mimikatz` এবং সন্দেহজনক PowerShell কমান্ডের মতো কীওয়ার্ড থাকবে।
এগুলি ফলস পজিটিভ, তাই hayabusa চালানোর অনুমতি দিতে আপনার সিকিউরিটি পণ্যে এক্সক্লুশন কনফিগার করতে হবে।
যদি আপনি ম্যালওয়্যার বা সাপ্লাই চেইন আক্রমণ নিয়ে চিন্তিত হন, তাহলে অনুগ্রহ করে hayabusa সোর্স কোড পরীক্ষা করুন এবং নিজেই বাইনারিগুলি কম্পাইল করুন।

Windows Defender-এর রিয়েল-টাইম প্রোটেকশনের কারণে আপনি বিশেষত রিবুটের পর প্রথম রানে ধীর রানটাইম অনুভব করতে পারেন।
আপনি সাময়িকভাবে রিয়েল-টাইম প্রোটেকশন বন্ধ করে অথবা hayabusa রানটাইম ডিরেক্টরিতে একটি এক্সক্লুশন যোগ করে এটি এড়াতে পারেন।
(এগুলি করার আগে অনুগ্রহ করে সিকিউরিটি ঝুঁকিগুলি বিবেচনা করুন।)

## Windows

একটি Command/PowerShell Prompt বা Windows Terminal-এ, শুধু উপযুক্ত 32-bit বা 64-bit Windows বাইনারিটি চালান।

### পাথে স্পেস থাকা কোনো ফাইল বা ডিরেক্টরি স্ক্যান করার চেষ্টা করার সময় ত্রুটি

Windows-এ বিল্ট-ইন Command বা PowerShell prompt ব্যবহার করার সময়, যদি আপনার ফাইল বা ডিরেক্টরি পাথে কোনো স্পেস থাকে তাহলে আপনি একটি ত্রুটি পেতে পারেন যে Hayabusa কোনো .evtx ফাইল লোড করতে পারেনি।
.evtx ফাইলগুলি সঠিকভাবে লোড করতে, নিশ্চিত করুন যে আপনি নিম্নলিখিতগুলি করছেন:

1. ফাইল বা ডিরেক্টরি পাথটি ডাবল কোট দিয়ে আবদ্ধ করুন।
2. যদি এটি একটি ডিরেক্টরি পাথ হয়, তাহলে নিশ্চিত করুন যে শেষ অক্ষর হিসেবে আপনি একটি ব্যাকস্ল্যাশ অন্তর্ভুক্ত করছেন না।

### অক্ষরগুলি সঠিকভাবে প্রদর্শিত হচ্ছে না

Windows-এ ডিফল্ট ফন্ট `Lucida Console` দিয়ে, লোগো এবং টেবিলে ব্যবহৃত বিভিন্ন অক্ষর সঠিকভাবে প্রদর্শিত হবে না।
এটি ঠিক করতে আপনার ফন্ট পরিবর্তন করে `Consalas` করা উচিত।

এটি ক্লোজিং মেসেজে জাপানি অক্ষরের প্রদর্শন ছাড়া বেশিরভাগ টেক্সট রেন্ডারিং ঠিক করবে:

![Mojibake](../assets/screenshots/Mojibake.png)

এটি ঠিক করতে আপনার কাছে চারটি বিকল্প রয়েছে:

1. Command বা PowerShell prompt-এর পরিবর্তে [Windows Terminal](https://learn.microsoft.com/en-us/windows/terminal/) ব্যবহার করুন। (প্রস্তাবিত)
2. `MS Gothic` ফন্ট ব্যবহার করুন। লক্ষ্য করুন যে ব্যাকস্ল্যাশগুলি ইয়েন সিম্বলে পরিণত হবে।
   ![MojibakeFix](../assets/screenshots/MojibakeFix.png)
3. [HackGen](https://github.com/yuru7/HackGen/releases) ফন্টগুলি ইনস্টল করুন এবং `HackGen Console NF` ব্যবহার করুন।
4. জাপানি অন্তর্ভুক্ত ক্লোজিং মেসেজগুলি প্রদর্শন না করতে `-q, --quiet` ব্যবহার করুন।

## Linux

আপনাকে প্রথমে বাইনারিটি এক্সিকিউটেবল করতে হবে।

```bash
chmod +x ./hayabusa
```

তারপর Hayabusa রুট ডিরেক্টরি থেকে এটি চালান:

```bash
./hayabusa
```

## macOS

Terminal বা iTerm2 থেকে, আপনাকে প্রথমে বাইনারিটি এক্সিকিউটেবল করতে হবে।

```bash
chmod +x ./hayabusa
```

তারপর, Hayabusa রুট ডিরেক্টরি থেকে এটি চালানোর চেষ্টা করুন:

```bash
./hayabusa
```

macOS-এর সর্বশেষ সংস্করণে, এটি চালানোর চেষ্টা করার সময় আপনি নিম্নলিখিত সিকিউরিটি ত্রুটিটি পেতে পারেন:

![Mac Error 1 EN](../assets/screenshots/MacOS-RunError-1-EN.png)

"Cancel" ক্লিক করুন এবং তারপর System Preferences থেকে, "Security & Privacy" খুলুন এবং General ট্যাব থেকে, "Allow Anyway" ক্লিক করুন।

![Mac Error 2 EN](../assets/screenshots/MacOS-RunError-2-EN.png)

এরপর, আবার এটি চালানোর চেষ্টা করুন।

```bash
./hayabusa
```

নিম্নলিখিত সতর্কবার্তাটি পপ আপ হবে, তাই অনুগ্রহ করে "Open" ক্লিক করুন।

![Mac Error 3 EN](../assets/screenshots/MacOS-RunError-3-EN.png)

আপনি এখন hayabusa চালাতে সক্ষম হওয়া উচিত।
